# Workflow 02 — Extract Techniques

## Goal

从书籍或小说中提炼可复用写作技巧，而不是总结内容。

## Required Outputs Per Source

Create these files under `library/<source-slug>/`:

```text
book-card.md
technique-bank.md
logic-model.md
style-card.md
chapter-retention.md
```

## Analysis Order

1. **Reader Promise** — 读者为什么愿意继续看。
2. **Opening Hook** — 开头如何制造问题。
3. **Mainline** — 主角目标、阻力、代价。
4. **Character Engine** — 欲望、恐惧、隐瞒、代价。
5. **Conflict Engine** — 冲突如何持续生产。
6. **Causality** — 选择如何带来后果。
7. **Chapter Retention** — 每章结尾如何留人。
8. **Emotional Mechanism** — 情绪不是怎么说，而是怎么发生。
9. **Human Texture** — 哪些细节去 AI。
10. **Reusable Rules** — 转成用户能直接写作的规则。

## Technique Extraction Rule

Each technique must be written as:

```text
技术名：
解决什么问题：
适用场景：
操作步骤：
失败形态：
中文商业小说改写版：
原创示例：
```

## Forbidden

- 不做读书笔记式流水总结。
- 不摘抄大段原文。
- 不把剧情当技巧。
- 不用“文笔很好、节奏不错、人物鲜明”这种空话。
- 不写无法执行的感想。
