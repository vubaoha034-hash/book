# Workflow 03 — Apply Technique Library To User Draft

## Goal

用技巧库诊断、修复、重写用户小说。

## Steps

1. Identify draft type:

```text
短小说 / 长篇章节 / 短剧脚本 / 抖音漫画脚本 / 简介 / 大纲 / 开头
```

2. Load relevant rules:

```text
rules/no-ai-smell.md
rules/novel-logic-checklist.md
library/global-technique-bank.md, if exists
library/global-anti-ai-rules.md, if exists
library/global-plot-logic.md, if exists
```

3. Diagnose with score.
4. Identify fatal issues.
5. Produce repair strategy.
6. Rewrite.
7. Self-check after rewrite:

```text
前三秒是否有钩子？
前十秒是否知道主角处境？
主线是否一句话说清？
冲突是否有现实代价？
是否删除了空泛解释？
人物是否有私心和不完美反应？
每段是否有动作/选择/信息变化？
结尾是否让人想评论？
```

## Rewrite Output Structure

```text
先给诊断：
再给重写策略：
最后给完整重写：
```

如果用户只要成稿，直接给重写稿，但仍要在内部完成诊断。
