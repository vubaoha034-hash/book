# Novel Logic Checklist

## 一句话主线

任何故事必须能回答：

```text
主角想要什么？
谁阻止他？
如果失败会失去什么？
为什么必须现在解决？
```

如果这四个问题答不出来，说明主线不清。

## 场景检查

每个场景必须有变化：

```text
进入场景前：主角以为 A。
场景结束后：主角发现 B / 失去 C / 被迫做 D。
```

如果场景结束后什么都没变，这场可以删。

## 冲突检查

好冲突不是“他们吵架”，而是：

```text
两个人都想保住某个东西，但这两个目标不能同时成立。
```

冲突必须落在现实代价上：

- 钱。
- 时间。
- 面子。
- 孩子。
- 身份。
- 亲情。
- 身体疼痛。
- 工作损失。
- 秘密曝光。
- 关系破裂。

## 反转检查

反转必须满足：

1. 前文埋过线。
2. 反转后重新解释前文。
3. 反转带来新代价。
4. 不能只为了吓读者。

坏反转：突然出现一个新设定解决问题。

好反转：读者回头一想，发现前面那个小动作早就在提示。

## 人物动机检查

每个重要人物都要有：

```text
想要：
害怕：
隐瞒：
误解：
底线：
代价：
```

人物不能只承担剧情功能。反派也要有现实理由，但不能用旁白洗白。

## 留存检查

每 300–500 字至少出现一次变化：

- 新信息。
- 更大压力。
- 误会升级。
- 反转。
- 选择。
- 代价。
- 情绪奖励。
- 关系变化。

章节结尾必须留下：

```text
新问题 / 新危机 / 新羞辱 / 新选择 / 新秘密 / 新反转
```

## 常见逻辑 bug

- 主角明明有更简单的方法却不用。
- 角色知道不该知道的信息。
- 道具突然出现。
- 危机太容易解决。
- 情绪恢复太快。
- 关系变化没有代价。
- 反派突然降智。
- 主角突然开挂。
- 时间线无法成立。
- 人物行为只服务作者想要的反转。

## 最终检查

```text
第一句是否有钩子？
前十秒是否知道主角困境？
主线是否一句话说清？
每场是否有变化？
冲突是否具体？
反转是否有铺垫？
人物选择是否符合动机？
情绪是否从动作里出来？
结尾是否让人想继续看或评论？
```
